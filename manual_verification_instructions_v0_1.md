# v0.1 実機確認手順

この文書は、AIシナリオ駆動型スマホWeb RPGエンジン v0.1 を実機（スマートフォンやタブレット）で確認するための手順をまとめたものです。エンジンには起動時バリデータ機能が実装されており、壊れたデータを検出した場合はゲーム開始を停止してエラー内容を表示します。

## 前提

* Python 3 がインストールされた PC または Mac を用意し、本パッケージのファイル一式を配置しておきます。
* スマートフォン／タブレットは PC と同じローカルネットワークに接続されている必要があります。
* 以下ではホスト PC の IP アドレスを `HOST_IP`、指定したポート番号を `PORT` と表記します。

## 準備

1. ターミナルまたはコマンドプロンプトを開き、パッケージを展開したディレクトリに移動します。
2. 次のコマンドで簡易 HTTP サーバーを起動します。

   ```bash
   python3 -m http.server 1420
   ```

   他のポートを使用したい場合は `1420` を任意の番号に変更してください。その場合、以下の URL も合わせて読み替えてください。

3. 起動したサーバーのホスト名／IP アドレス（例: `192.168.1.5`）とポート番号を確認します。この情報はスマートフォンからアクセスする際に使用します。

## 通常パックの確認

1. スマートフォンのブラウザで次の URL を開きます（`HOST_IP` と `PORT` を実際の値に置き換えてください）。

   ```
   http://HOST_IP:PORT/index_compiled.html
   ```

2. ゲームがロードされ、タイトル画面やロード画面のあとにフィールドが表示されることを確認します。仮想十字キーや「決定」ボタンが画面に表示されるはずです。

3. D‑パッドや画面タップ、外付けキーボード（PC で動作確認する場合）を使って主人公を移動し、村長 NPC に近付きます。近づくと「話す」というラベルが表示されます。決定ボタンを押して会話が始まり、複数行のセリフが表示されることを確認してください。

4. 会話が終了すると `chief_talked` フラグが保存され、再度村長に話しかけた際には別のセリフが表示されることを確認できます。

5. コンソールエラーやページエラーが発生していないか（ブラウザのデベロッパーツールを開ける場合は確認してください）、ゲームが途中で停止しないことを確認します。

## 壊れた Content Pack の確認（Validator エラー）

1. 上記と同じサーバーが稼働している状態で、スマートフォンのブラウザで次の URL を開きます。

   ```
   http://HOST_IP:PORT/index_compiled.html?pack=sample_broken_missing_dialogue_pack
   ```

2. ゲーム本編（Canvas や仮想十字キー）は表示されず、白地に黒文字の「Content Pack検証エラー」画面が表示されることを確認します。この画面には以下の情報が含まれます。

   * `検証エラーのためゲームを開始できません`
   * `error: 1`, `warning: 0`
   * `[error] 存在しない会話ID参照`
   * `イベント「event_talk_chief_first」が、存在しない会話ID「dialogue_missing_for_validator_test」を参照しています。`
   * `対象ファイル: events.json`
   * `対象イベントID: event_talk_chief_first`
   * `問題ID: dialogue_missing_for_validator_test`
   * `対処` として dialogues.json に ID を追加するか events.json を修正する旨のメッセージ

3. 画面にゲーム本編のキャンバスや仮想十字キーが表示されていないことを確認します。また、ゲームが操作できない（開始されていない）ことも確認してください。

4. ブラウザのコンソールに追加の JavaScript エラーが出ていないか、ネットワークタブで 404/5xx 応答が発生していないかを確認すると、Validator が起動時に正しく処理していることがわかります。

## ファイル構成

実機確認用パッケージには以下のファイル／ディレクトリが含まれています。

```
index_compiled.html        ... コンパイル済みの HTML エントリポイント
compiled/main.js           ... TypeScript をトランスパイルしたゲーム本体
src/styles/global.css      ... 共通スタイルシート
src/styles/mobile-ui.css   ... モバイル UI 用スタイルシート
content/sample_minimal_pack/   ... 通常の Content Pack
content/sample_broken_missing_dialogue_pack/   ... 壊れた Content Pack（Validator エラー用）
```

## 注意事項

* サーバーを起動するポートが他の用途で使われている場合は別のポート番号を指定してください。
* スマートフォンからアクセスできない場合、ファイアウォールやネットワーク設定を確認してください。
* 上記の手順は基本的な動作確認用です。さらに詳細な検証を行う場合は、Playwright のような自動化ツールやデベロッパーツールを併用してください。
